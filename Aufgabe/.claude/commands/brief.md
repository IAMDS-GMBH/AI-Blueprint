# /brief – Executive Status Brief

Erstelle eine kompakte Zusammenfassung des aktuellen Projektstatus fuer Management / CEO.

## Ablauf

1. git log der letzten 7 Tage lesen
2. Offene Issues / PRs analysieren (via GitHub MCP falls verfuegbar)
3. tasks/todo.md lesen (offene Punkte)
4. tasks/lessons.md lesen (aktuelle Problemmuster)
5. Brief erstellen

## Output-Format

```
## Projekt-Brief: [PROJEKTNAME]
Stand: [DATUM]

### Status
[ROT / GELB / GRUEN] – [1 Satz Begruendung]

### Was wurde erledigt (letzte Woche)
- ...

### Was ist blockiert / kritisch
- ...

### Naechste Schritte (diese Woche)
- ...

### Entscheidung benoetigt
- [Falls nichts: "Keine offenen Entscheidungen"]
```

## Constraints
- Max. 1 Seite
- Kein Tech-Jargon wenn nicht noetig
- Immer mit konkreter Handlungsempfehlung schliessen
