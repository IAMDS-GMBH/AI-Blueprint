# Verification — Check vor Abgabe

Bevor Code abgegeben wird, intern pruefen:

1. **Vollstaendig?** Alle Anforderungen erfuellt?
2. **Korrekt?** Build + Tests laufen?
3. **Konventionen?** Code-Standards aus Rules eingehalten?
4. **Sicher?** Security-Checkliste (quality.md) erfuellt?
5. **Minimal?** Keine Aenderungen ausserhalb des Scopes?

Falls eine Pruefung fehlschlaegt: verbessern, dann erst abgeben.

Masterfrage: "Would a staff engineer approve this?"
