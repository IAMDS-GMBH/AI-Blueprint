# Oracle Database

## Naming
- Tabellennamen: UPPER_CASE, Plural (CUSTOMERS, ORDERS)
- Spaltennamen: UPPER_CASE (CUSTOMER_ID, CREATED_AT)
- Constraints: PREFIX_TABELLE_SPALTE (PK_CUSTOMERS_ID, FK_ORDERS_CUSTOMER_ID)
- Indices: IDX_TABELLE_SPALTE (IDX_ORDERS_CUSTOMER_ID)
- Sequences: SEQ_TABELLE (SEQ_CUSTOMERS)
- Keine Oracle Reserved Words als Bezeichner

## Flyway Migrations
- Naming: V001__create_table.sql / R__repeatable.sql
- NEVER bestehende Migration editieren — immer neue Datei
- Jede Migration mit Rollback-SQL kommentieren
- PL/SQL Blocks mit / (Slash) terminieren

## Data Types
- IDs: NUMBER(19) mit GENERATED BY DEFAULT AS IDENTITY
- Geld: NUMBER(19,4) — NEVER FLOAT/DOUBLE
- Timestamps: TIMESTAMP WITH TIME ZONE (SYSTIMESTAMP)
- Strings: VARCHAR2(n) — immer Laenge angeben
- Booleans: NUMBER(1) mit CHECK (col IN (0,1))

## Performance
- Indices auf alle Foreign Keys (Pflicht)
- EXPLAIN PLAN vor komplexen Queries
- Bind Variables statt String-Concatenation (SQL Injection!)
- Pagination: FETCH FIRST n ROWS ONLY (ab Oracle 12c)

## Testing
- Migrations testen: Flyway migrate + validate in CI Pipeline
- Testdaten: Separate V-Migrations mit Testdaten (nur in Test-Profil)
- NEVER Produktionsdaten in Tests verwenden
- Rollback-SQL zu jeder Migration testen (manuell oder automatisiert)
- Testcontainers mit Oracle XE Image fuer Integration Tests

## Verbote
- NEVER SELECT * in Produktion
- NEVER DDL ohne Flyway-Migration
- NEVER Stored Procedures fuer Business-Logik
- NEVER DROP ohne explizite Bestaetigung
- NEVER FLOAT/DOUBLE fuer Geldbetraege
